## pip
### 定位
官方的项目依赖管理工具，主要为具体项目安装第三方依赖，会把包安装到当前激活的虚拟环境（venv）或者全局环境中
### 痛点
**速度慢**：纯 Python 实现，解析依赖和下载的速度相对较慢。
**不隔离**：如果你不用虚拟环境，直接 `pip install` 到全局，不同的项目之间容易发生依赖冲突（比如项目 A 需要 `requests==2.0`，项目 B 需要 `requests==3.0`）。
**功能单一**：只管装包，不管虚拟环境的创建（需要配合 `python -m venv` 使用）。
### 典型命令
```bash
pip install requests
```
## pipx
### 定位
专门用于安装和运行python命令行应用的工具，安装那些在终端全局使用的工具，比如代码格式化工具`black`,`ruff`。它的优势是具有隔离性，会为每一个安装的应用创建独立的虚拟环境，然后把可执行文件链接到全局PATH（如 ~/.local/bin）中，同时能解决全局依赖冲突的问题。
### 注意点
只针对可执行工具，不适合用来管理项目依赖
### 典型命令
```bash
pipx install httpie
```
## uv
### 定位
用rust编写的极速、全能型python包与项目管理工具，他的安装与解析速度比pip快10到100倍。

### 替代命令
**pip**：`uv pip install requests`
**pipx**: `uv tool install black` (同样自带虚拟环境)
**venv**: `uv venv` (秒建虚拟环境)
**pyenv**: `uv python install 3.12` (自动下载与管理python版本)

### 典型命令
```bash
# 安装项目依赖
uv pip install -r requirment.txt
# 安装全局工具
uv tool install ruff
# 运行一次性脚本，包括自动处理依赖
uv run --with requests script.py
```
