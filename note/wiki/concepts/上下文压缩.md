# 上下文压缩

在 prompt / 工具输出 / 对话历史抵达 LLM **之前**将其压缩，降低 token 消耗、费用、上下文占用，同时保持回答准确率。

## 为什么需要

`claude code`、`codex` 等 agent 工具输出庞大且冗余 → 上下文填满、费用上升、有效记忆缩短。

## 代表实现

[[Headroom]]：本地运行的压缩代理，可逆、支持按需检索，压缩率 60%–95%。

## 同谱系（agent 效率）

| 方向 | 机制 | 代表 |
|---|---|---|
| 压缩 | 拦截 + 变换消息流 | [[Headroom]] |
| 固化 | 把高频操作沉淀为技能 / 记忆 | [[技能自进化闭环]]（SkillOpt） |

> [!key-insight] 互补不互斥
> 压缩省的是「输入带宽」，技能固化省的是「重复推理」。两者可叠加。
