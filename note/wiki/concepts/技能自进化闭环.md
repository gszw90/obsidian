# 技能自进化闭环

像训练模型一样训练 agent 的 skill/memory：离线重放历史会话，用留出集（held-out）分数做门控，只保留确有提升的规则改动。代表实现 [[SkillOpt]]。

## 六步闭环

| 步骤 | 作用 |
|---|---|
| Harvest | 收集历史会话 |
| Mine | 挖掘反复出错/重复的任务 |
| Replay | 离线重跑任务（baseline vs candidate） |
| Reflect | 分析失败，生成规则修改提案 |
| Gate | 留出集分数提升才保留 |
| Stage & Adopt | 暂存提案 → 人工审核 → 采纳 |

## 关键设计

- **留出集门控** — 防止过拟合 / 退化的改动被采纳
- **staging 分离** — 线上文件只在 `adopt` 时改，且自动备份
- **增量 harvest** — 每夜只看新增会话（`last_harvest` 时间戳）

## 陷阱（实测）

- 噪声会话（observer / MCP 日志）会污染 mine → 产出垃圾规则，gate 可能误通过
- `mock` 后端确定性重放，无法真正改进；需 `claude` / `codex` 后端
