# 技能自进化闭环

像训练模型一样训练 agent 的 skill/memory：离线重放历史会话，用留出集（held-out）分数做门控，只保留确有提升的规则改动。代表实现 [[SkillOpt]]。

## 六步闭环

| 步骤 | 作用 |
|---|---|
| Harvest | 收集历史会话 |
| Mine | 挖掘反复出错/重复的任务 |
| Replay | 离线重跑任务（baseline vs candidate） |
| Reflect | 分析失败，生成规则修改提案 |
| Gate | 留出集分数提升才保留 |
| Stage & Adopt | 暂存提案 → 人工审核 → 采纳 |

## 关键设计

- **留出集门控** — 防止过拟合 / 退化的改动被采纳
- **staging 分离** — 线上文件只在 `adopt` 时改，且自动备份
- **增量 harvest** — 每夜只看新增会话（`last_harvest` 时间戳）

## 陷阱（实测）

- 噪声会话（observer / MCP 日志）会污染 mine → 产出垃圾规则，gate 可能误通过
- `mock` 后端确定性重放，无法真正改进；需 `claude` / `codex` 后端

### 陷阱 3（2026-07-03 实证）：rule judge 自证循环

`llm_miner` 默认产出 `reference_kind="rule"` 任务 → 判分走本地 `judges.py`（regex / contains / section_present），**不调 LLM**。`--backend claude` 只改 *生成*，不改 *判分*。本 vault 实测：

| 实验 | judge | held-out | gate | 真伪 |
|---|---|---|---|---|
| night 6 | rule（pattern-match） | 0.050 → 0.550 | accept | **假阳性** |
| night 7 | claude rubric（LLM） | 0.017 → 0.017 | reject | 真值 |

night 6 候选规则全是「输出必须含 `## Pages created` 表格 / 字面 `Cross-reference` / `.obsidian/.gitignore`」——其 `_why` 字段自承「强制满足 evaluator grep 的串」。对测试 harness 过拟合，非真技能。换真 LLM judge（night 7）立刻打回：分数原地不动。

> [!key-insight] rule judge 奖励「输出含 evaluator 检查的字符串」
> 循环。metric 不是质量，是格式匹配度。gate 通过 ≠ 真改进。

## 正确验证方法（本 vault 实证）

默认 miner 偏向 rule judge → 无法真验证。绕过：手写 `reference_kind="rubric"` 任务，`--tasks-file` 注入，配 `--backend claude` → 走 `ClaudeBackend.judge()`（真 LLM 打分）。

```bash
SKILLOPT_SLEEP_REPO=/home/zeng/projects/python/SkillOpt \
  sleep.sh run --backend claude \
  --tasks-file <rubric_tasks.json> --project "$(pwd)" --scope invoked
```

- rubric 任务：`reference_kind="rubric"`，`reference` 写「好答案满足什么」（真实质量标准，非字符串匹配）
- split：标 `train` / `test`，留出（test）用于 gate
- 判分路径：`replay.py` 对 rubric 任务调 `backend.judge()`，不走本地 `score_rule_judge`
- 真验证特征：分数会 **持平或下降**（reject），不膨胀。night 7 的 0.017→0.017 即此信号

> [!warning] `--backend claude` ≠ 真验证
> 默认任务仍走 rule judge。必须 `--tasks-file` 注入 rubric 任务才走 LLM 判分。
